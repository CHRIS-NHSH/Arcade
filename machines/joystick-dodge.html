搖桿躲避機

在 OLED 螢幕上操控一個小方塊，躲開不斷從畫面右側冒出、往左飛來的障礙物。這個專案的重點是「即時碰撞判定」與「難度隨時間遞增」的做法。

難度★★★☆☆
製作時間約 3-4 小時
核心零件SSD1306 OLED + 搖桿

遊戲玩法

玩家角色固定在畫面左側，只能上下移動。障礙物以固定間隔從右側生成，往左等速移動。存活時間越長分數越高，且每存活 10 秒障礙物生成速度會加快一階。

所需材料

零件數量備註

SSD1306 0.96" OLED（I2C）1128x64 解析度
類比搖桿模組1只需 Y 軸

函式庫

建議使用 Adafruit_SSD1306 搭配 Adafruit_GFX，繪圖與畫矩形/文字的 API 都很直覺。

核心程式邏輯

用 millis() 取代 delay() 來同時處理「畫面更新」與「障礙物生成」兩件不同頻率的事，是這個專案最關鍵的技巧。

int playerY = 32;
int obstacleX[5], obstacleY[5];
unsigned long lastSpawn = 0;
int spawnInterval = 1200; // 毫秒，會隨時間縮短

void loop() {
  playerY = map(analogRead(A0), 0, 1023, 0, 64 - PLAYER_H);

  if (millis() - lastSpawn > spawnInterval) {
    spawnObstacle();
    lastSpawn = millis();
    if (spawnInterval > 400) spawnInterval -= 20; // 難度遞增
  }

  moveObstacles();
  if (checkCollision(playerY)) { gameOver(); return; }

  drawFrame(playerY);
}

延伸挑戰

  • 加入「無敵道具」：吃到後短暫可穿過障礙物。
  • 障礙物形狀多樣化，並依難度調整出現機率。
  • 加入最高分記錄，用 EEPROM 儲存。
AI 協作提示

碰撞判定的數學（矩形是否重疊）常常一開始寫得太寬鬆或太嚴格，導致「明明沒撞到卻死了」。把你的 checkCollision() 函式與座標定義告訴 AI，請它畫出示意圖或用文字描述判斷邏輯，很快就能抓到問題。
