聖誕樹互動遊戲機

把一整棵聖誕樹外殼變成雙人對戰機台：17 顆可定址 LED 當裝飾燈，6 顆按鈕分成左右兩側，同一套硬體可以切換玩 5 款完全不同的小遊戲。這台的重點是用狀態機管理「選單＋5款遊戲」這種多模式切換的架構。

難度★★★★☆
製作時間約 6-8 小時（含外殼與燈珠黏貼）
核心零件17 顆 WS2812 可定址 LED + 6 顆按鈕

整體設計：一棵樹，五款遊戲

樹尖是第 0 號 LED，稱為「星星」，負責顯示選單、題目與勝負訊號；樹身其餘 16 顆 LED 依左右分成 P1、P2 各 8 顆。底座控制台每位玩家各有紅、綠、藍 3 顆按鈕。開機後樹身跑彩虹燈效，接著進入選單，用 P1 的按鈕選遊戲、確認後進入對應關卡，玩家對戰到其中一方燈全亮就分出勝負，結束後自動回到選單。

選單操作

按鈕選單模式功能
P1 紅色鈕切換到上一個遊戲（星星顯示該遊戲的代表色）
P1 綠色鈕切換到下一個遊戲
P1 藍色鈕確認選擇，星星顏色「流滿」整棵樹後進入遊戲

五款遊戲一覽

關卡名稱玩法一句話
紅關搶救聖誕裝飾星星隨機亮紅或綠，看到對應顏色搶按同色按鈕，先馳得點
橙關爬煙囪拔河瘋狂連打任一按鈕，自己那側的燈從樹根往上爬，先爬到頂端獲勝
黃關聖誕記憶星星閃出一段紅／藍序列，燈熄滅後照原順序按對應按鈕，序列會越來越長
綠關點亮星星跑馬燈忽快忽慢地跑向樹頂，輪流搶在燈剛好跑到星星的瞬間按下按鈕
藍關聖誕色彩大師星星顯示目標顏色，同時按下對應的顏色組合（如紅+藍＝紫）配出正確混色

所需材料

零件數量備註

杜邦線、麵包板若干

接線提醒

程式用 INPUT_PULLUP 讀取按鈕，接線時按鈕一端接對應腳位、另一端直接接 GND 即可，不需要外接下拉電阻。17 顆 LED 全部全白全亮時瞬間電流不小，建議用外部 5V 電源供電，只與 Arduino 共用 GND。

核心程式邏輯：用狀態機管理「選單＋5款遊戲」

整台機器只有一個 enum GameState，loop() 裡用 switch 決定現在該執行選單邏輯還是某一款遊戲的邏輯，遊戲之間切換只是改變這個狀態值，硬體讀取與燈光繪製的寫法可以在 5 款遊戲之間重複利用。

enum GameState {
  MENU,
  GAME_1_RED, GAME_2_ORANGE, GAME_3_YELLOW, GAME_4_GREEN, GAME_5_BLUE
};
GameState currentState = MENU;

void loop() {
  P1_R.update(); P1_G.update(); P1_B.update();
  P2_R.update(); P2_G.update(); P2_B.update();

  switch (currentState) {
    case MENU:          runMenu(); break;
    case GAME_1_RED:    runGame1_WhackAMole(); break;
    case GAME_2_ORANGE: runGame2_TugOfWar(); break;
    case GAME_3_YELLOW: runGame3_Memory(); break;
    case GAME_4_GREEN:  runGame4_Timing(); break;
    case GAME_5_BLUE:   runGame5_ColorMix(); break;
  }
}

紅關「搶救聖誕裝飾」是最單純的一款，示範了「隨機出題 → 監聽按鍵 → 判定搶答 → 更新計分燈」的基本流程，其餘 4 款都是同一套流程的變化版：

void runGame1_WhackAMole() {
  if (gameInit) {                 // 每次進入遊戲先重置狀態
    p1Score = 0; p2Score = 0;
    strip.clear();
    gameInit = false;
    g1_waiting = true;
    g1_waitStart = millis();
  }

  if (g1_waiting) {
    if (millis() - g1_waitStart > random(1000, 3000)) {  // 隨機延遲出題
      g1_targetColor = random(2);
      strip.setPixelColor(STAR_LED, g1_targetColor == 0 ? COL_RED : COL_GREEN);
      strip.show();
      g1_waiting = false;
    }
  } else {
    bool p1_hit = (g1_targetColor == 0 && P1_R.pressed()) || (g1_targetColor == 1 && P1_G.pressed());
    bool p2_hit = (g1_targetColor == 0 && P2_R.pressed()) || (g1_targetColor == 1 && P2_G.pressed());
    if (p1_hit || p2_hit) {
      if (p1_hit) { p1Score++; winAnim(1); } else { p2Score++; winAnim(2); }
      updateScoreboard();
      if (p1Score >= LEDS_PER_SIDE || p2Score >= LEDS_PER_SIDE) gameOver(p1Score > p2Score ? 1 : 2);
      else { g1_waiting = true; g1_waitStart = millis(); }
    }
  }
}

延伸挑戰

  • 加入單人模式，讓其中一側改成電腦自動出招，方便一個人也能練習。
  • 用 DFPlayer Mini 播放每款遊戲各自的背景音樂與語音提示，取代單純蜂鳴器。
  • 用 EEPROM 記錄 5 款遊戲各自的歷代最高分或最快紀錄。
  • 樹身改用 WS2812B 燈條纏繞，燈數更多、氣氛效果更強。
AI 協作提示

這台機器最值得跟 AI 討論的地方，就是「選單 + 5 款遊戲」這種多模式狀態機的架構本身：把 5 款遊戲的規則、勝負條件先整理成表格（可以參考 AI 協作教學的做法），再請 AI 幫你檢查狀態切換有沒有漏掉的分支，之後要再加入第 6 款遊戲時也會比較容易維護。
