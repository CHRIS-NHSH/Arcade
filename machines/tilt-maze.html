傾斜迷宮機

用陀螺儀／加速度計偵測機台傾斜角度，控制畫面上的球滾出迷宮。這個專案會讓你第一次體會到「感測器訊號雜訊」與濾波處理的重要性。

難度★★★☆☆
製作時間約 3-4 小時
核心零件MPU6050 + OLED

遊戲玩法

玩家實際傾斜整個機台外殼，畫面上的球會依傾斜方向加速滾動。目標是在時間限制內把球滾到終點，途中若掉進畫在迷宮裡的陷阱格就必須重來。

I2C 位址提醒

MPU6050 與 OLED 都是 I2C 裝置，各自有固定位址（通常是 0x68 與 0x3C），可以共用 A4/A5 兩條線，寫程式時用各自的位址溝通即可，不需要額外接線。

核心程式邏輯

陀螺儀讀值會有雜訊，直接拿來當速度使用球會抖動，實務上會用「低通濾波」讓數值平滑：

float velX = 0, velY = 0;
float ballX = 64, ballY = 32;

void loop() {
  float rawAx = readAccelX(); // -1.0 ~ 1.0，代表傾斜方向與程度
  float rawAy = readAccelY();

  // 低通濾波：新舊數值加權平均，減少抖動
  velX = velX * 0.8 + rawAx * 0.2;
  velY = velY * 0.8 + rawAy * 0.2;

  ballX += velX * 2;
  ballY += velY * 2;

  if (hitWall(ballX, ballY)) { velX = 0; velY = 0; } // 撞牆停止
  if (hitTrap(ballX, ballY)) resetBall();
  if (reachedGoal(ballX, ballY)) showWin();

  drawMaze(ballX, ballY);
}

延伸挑戰

  • 設計多張迷宮地圖，用陣列儲存牆壁座標，過關後自動切換。
  • 加入計時挑戰模式，顯示破關所需秒數。
  • 把 OLED 換成實體滾珠迷宮外殼，Arduino 只負責計時與過關判定。
AI 協作提示

濾波係數（例如上面的 0.8 / 0.2）要調到手感剛好，很花時間試誤。可以請 AI 解釋低通濾波公式背後的原理，並依你想要的「靈敏度 vs 穩定度」描述，幫你估算合理的起始參數再實測微調。
