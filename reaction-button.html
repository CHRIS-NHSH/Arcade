反應力搶答機

多人搶答＋反應速度測試二合一。核心是學會用「中斷（interrupt）」精準判定誰先按下按鈕，做出不會有爭議的裁判機。

難度★☆☆☆☆
製作時間約 1-2 小時
核心零件按鈕 + 七段顯示器

遊戲玩法

模式一「搶答賽」：主持人按下開始鍵後，最先按下自己按鈕的玩家亮燈並鎖住其他人的按鈕。模式二「反應測試」：LED 隨機延遲後亮起，玩家看到燈亮立刻按鍵，機台顯示反應時間（毫秒）。

為什麼用中斷

Uno 的 D2、D3 支援硬體中斷，按鈕一按下就會立刻打斷主程式去記錄時間，不會受到 loop() 裡其他運算延遲影響，判定最公平。其餘按鈕可用輪詢，誤差在毫秒等級通常可接受。

核心程式邏輯

volatile bool locked = false;
volatile int winner = -1;
volatile unsigned long pressTime = 0;

void onPlayer1Press() { buzz(0); }
void onPlayer2Press() { buzz(1); }

void buzz(int player) {
  if (locked) return;      // 已經有人搶到，其餘按鍵忽略
  locked = true;
  winner = player;
  pressTime = millis();
}

void setup() {
  attachInterrupt(digitalPinToInterrupt(2), onPlayer1Press, FALLING);
  attachInterrupt(digitalPinToInterrupt(3), onPlayer2Press, FALLING);
}

void loop() {
  if (locked) {
    lightUpWinner(winner);
    showOnDisplay(winner + 1);
    delay(2000);
    resetRound();
  }
}

延伸挑戰

  • 加入「隨機延遲搶答」模式：太早按下算犯規。
  • 用 EEPROM 記錄歷史最快反應時間，開機後顯示紀錄。
  • 擴充到 6-8 人版本，改用矩陣掃描讀取按鈕，節省接腳。
AI 協作提示

中斷函式（ISR）裡能不能寫什麼、不能寫什麼常常搞混，例如不能在裡面用 delay()。這是很適合丟給 AI 檢查程式碼的地方——把你的 ISR 貼給 AI，請它抓出「ISR 裡不該出現」的寫法。
