用 AI 做出你的遊戲機台

如何用 AI 協助設計並做出一台像 機台教學庫 裡那樣好玩的 Arduino 遊戲機台。重點不在會不會叫 AI 生程式碼，而在想不想得清楚——一台好玩的機台，玩法、零件、回饋該怎麼配。

核心觀念 製作步驟 設計規格 提示詞產生器 好習慣 測試與展示

1核心觀念

與其叫 AI 從零生出一整台機台，不如先把設計想清楚，再請它實作程式碼。為什麼？因為 AI 生出來的程式碼「看起來能編譯、能燒錄」不代表電路跟玩法真的對——要做的，是把想法講清楚、再一一驗收。這頁要練的，是兩件帶得走的能力：

一、把設計想清楚，再開口問 AI。先寫好規格與接腳表，AI 才生得準（見下方規格）。

二、會驗收自己的成果。會用三用電表量、會看 Serial Monitor、會一個零件一個零件單獨測。

一句話：別只想著「叫 AI 生一份 .ino 給我」，而是先當設計師、再當工程師、最後當品管。這三件事換任何題目、任何板子都用得上。

2一台機台怎麼做出來

先看整個流程，後面幾段再一一細談。

  1. 玩 + 拆解：先玩現有機台或範例，拆解它的玩法、用了哪些零件、怎麼給玩家回饋——也就是下面要教的「規格」。
  2. 寫規格：把想做的機台，玩法、零件、接腳表填清楚——這一步先別碰 AI。
  3. 單元測試：先在麵包板上把每個零件（按鈕、LED、蜂鳴器、馬達…）各自接好、各自測過會動，再談整合。
  4. 用 AI 實作：把規格 + 技術要點用下面的產生器組成提示詞，貼給 AI，邊生、邊燒錄、邊測。
  5. 互測 + 展示：跟同學交換玩、回報問題、修好、再展示。機台本來就是要被別人玩的。

3設計規格

網站上的機台教學都遵循同一套結構。動手前先把這份規格想清楚，先別急著找 AI。

玩法是什麼？

玩家看到什麼、操作哪個零件（按鈕、搖桿、光敏電阻…）、要做什麼動作？

怎樣算過關／得分？

一個明確、不含糊、能用數字判斷的成功條件。

過關給什麼回饋？

LED 燈效、蜂鳴器音效、伺服馬達動作、LCD／OLED 顯示文字，擇一或組合。

零件與接腳？

用了哪些元件（Arduino Uno R4、按鈕、LED、蜂鳴器、伺服馬達、感測器…），各接哪一支腳位。這決定電路真正接不接得起來，最容易被忘記，卻最影響能不能動。

參數能調什麼？ 進階選用

參數是為了之後能快速調整難度（時間限制、分數門檻、感測器靈敏度），可以用可調電位器、序列埠指令，或 Uno R4 WiFi 版做網頁後台。對機台能不能玩不是必要的，先做出能玩的版本，行有餘力再加。

4規格 → 提示詞產生器

把規格填一填，下面的技術要點建議全勾（它們會附在提示最後），按「產生提示詞」就會組出一段可直接貼給 AI 的提示。這就是把設計變成 AI 看得懂的需求。

① 玩法（玩家操作哪個零件、做什麼動作） ② 怎樣算過關／得分 ③ 過關後給玩家的回饋 ④ 使用的零件與接腳 ⑤ 參數能調整什麼（進階選用，可留空） 技術要點：AI 常漏、建議全勾（會附在提示詞最後）

這幾條 AI 容易漏、漏了又不容易發現，每條都標了「沒寫會怎樣」。

5要養成的習慣

  • 驗證，不盲信。AI 給的程式碼，每個功能都自己實測：按鈕在 Serial Monitor 看得到訊號嗎？LED 真的會亮嗎？馬達會不會抖動或過熱？供電夠不夠？
  • 會回報問題。講「具體現象」而不是只說「壞了」。試試這個句型：
    「我按了＿＿，預期會＿＿，結果卻＿＿。」能附上 Serial Monitor 截圖或接線照片更好，這樣 AI 才修得準。
  • 需求講滿、工作切小。一開始把零件、接腳、規則一次講清楚，沒講的 AI 會自己猜；後續修改一次只改一個功能，改一段、燒錄一段、驗一段，才抓得到錯、保持得住掌控。這不只對 AI，做任何專題都適用。

6做好之後：測試與展示

機台做好之後，重點是能穩定被別人玩、也能被別人看懂你怎麼做的。兩件事都做，這個專題才算完整。

GitHub

保存程式碼與版本紀錄

把 .ino 程式碼、接線圖（或 Fritzing 檔）、零件清單放進同一個倉庫，改完重新上傳就有紀錄，之後想回溯某一版也找得到。

建倉庫 → 上傳程式碼與接線圖 → 寫一份 README.md 說明零件與玩法

→ 註冊 GitHub

展示影片／QR Code

讓沒帶機台在身邊的人也能看懂

錄一段 30–60 秒的操作示範，完整拍到「玩法 → 過關 → 回饋」。上傳後用 QR code 貼在機台旁，展示時掃碼就能看。

若是 Arduino Uno R4 WiFi 版，也可以做一個網頁後台即時顯示分數，再用同樣方式部署到 GitHub Pages。

怎麼選：先求「機台能穩定被玩」再談展示形式。程式碼上 GitHub、接線圖拍照存證、留一段展示影片，這三件事做齊，別人不用在場也能懂你做了什麼、怎麼做的。
